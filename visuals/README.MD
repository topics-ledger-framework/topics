# 🧪 TLF Interactive Visualizations

Welcome to the `visuals` directory. This folder contains a collection of interactive HTML/JavaScript simulations created during the foundational research phase of the **Topics Ledger Framework (TLF)**. 

## 📌 About These Simulations

Before the high-dimensional tensor computations were deployed on Google Cloud, these lightweight interactive modules were built to visually conceptualize the core mathematics of the framework. They serve as interactive sandboxes bridging abstract theory and visual intuition. 

By experimenting with these pages, you can physically see how algorithmic consensus generates time, gravity, and mass from a pure state of superposition (the Void).

## 🚀 How to Run

Because these are standalone, single-page web applications built with Vanilla JavaScript and Three.js, there are no complex dependencies, builds, or backend servers required.

1. **Clone or Download:** Pull this repository to your local machine.
2. **Open in Browser:** Simply double-click any `.html` file in this folder to run it in a modern web browser (Chrome, Firefox, Edge, Safari).
3. **Interact:** Use the on-screen HUD panels to adjust parameters, trigger genesis events, and manipulate the thermodynamic metrics in real-time.

## 📂 Simulation Index

### 1. Topic-Ledger: Void Boundaries (`void_walls.html`)
**Focus:** Genesis, Causal Strands, and Isomorphic Twins.
This 3D simulation models the birth of a "Cognitive Idea" from the Void. You can trigger a Genesis Block and watch as external contexts ("Minds") collide with the Topic, generating Causal Blocks along a glowing helical strand. 
* **Key Features:** Click "Reverse Engineer" to visually trigger a Grover Rewind, tracing the ledger backward to find causal isomorphisms and spawn evolutionary "Twins."

### 2. Topic-Ledger: The Singularity (`singularity-sim.html`)
**Focus:** Logarithmic Gravity, Event Horizons, and Mass Assimilation.
An extreme-scale evolution of the Void Boundaries simulation. This sandbox tests how the physics engine handles massive gravitational stakes. As the Topic absorbs context, an Event Horizon forms.
* **Key Features:** Introduces "Mass Assimilation" mechanics. When enough Twins are generated, the system consumes them to trigger an evolutionary leap, radically increasing the Singularity's density and IQ.

### 3. Topic-Ledger: Chronogenesis (`spiral.html`)
**Focus:** Time Generation and Algorithmic Dilation.
This visualizer strips away the complex physics to focus purely on the flow of Time. It demonstrates the axiom that time is not a straight line, but a minted spiral of consensus. 
* **Key Features:** Watch the "Cursor of Reality" mint blocks in real-time. By increasing the Stakes (Gravity) via the HUD, you can observe **Time Dilation**—the chain physically densifies and mints faster as mass increases.

### 4. Topic-Ledger: Multiverse Genesis (`sim-multiverse.html`)
**Focus:** Parent/Child Chains, Black Holes, and White Holes.
A 2D particle physics simulation modeling how distinct ledger chains interact. It features a "Parent Chain" (The Bulk) and a "Child Chain" (Our Universe).
* **Key Features:** Initiate a collapse in the Parent Chain to create a Black Hole. As it consumes context density (particles), you can trigger a "Fork" to transfer that accumulated mass through a wormhole—igniting a White Hole (Big Bang) in the Child Chain.

### 5. Topic-Ledger: Dark Matter & Strange Attractors (`sim-darkmatter.html`)
**Focus:** Dormant Context and Phase Space Gravity.
A 2D ecosystem visualizing the concept of "Dark Matter" within the ledger framework. In TLF, Dark Matter represents dormant legacy contexts from ancestor chains. 
* **Key Features:** Inject Dark Matter into the grid. While invisible, these nodes act as "Strange Attractors." You will watch them silently warp the gravitational flow of floating context (food), clumping information together without actively participating in the organism's ledger.

## 🛠️ Technology Stack

These visualizers are built strictly using standard web technologies: **HTML5 Canvas**, **Vanilla JavaScript**, and **Three.js** (for WebGL rendering). This architectural choice was made to ensure the underlying math remains completely transparent and accessible to anyone reviewing the TLF theory, without burying the logic behind heavy UI frameworks or complex build pipelines.
